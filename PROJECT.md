# 📚 Кулинарная книга PWA — Документация проекта

**Версия:** 25.16  
**Файл приложения:** `index.html` (~308 KB, ~6905 строк, всё в одном файле)  
**Тип:** Single-page PWA, чистый HTML/CSS/JS без фреймворков  
**Хранилище:** только `localStorage` браузера  

---

## 🗂 Структура файлов архива

```
cookbook-pwa-vXX.X.zip
├── index.html        ← всё приложение в одном файле
├── manifest.json     ← PWA манифест (иконки, название, цвета)
├── sw.js             ← Service Worker (офлайн-кэш)
└── PROJECT.md        ← этот файл (документация)
```

**Важно:** `index.html` содержит всё — HTML, CSS и JS (~161 функция). Никаких внешних зависимостей, никаких npm, никаких сборщиков.

## 🍽 Структура рецепта (объект в `db.recipes[]`)

```js
{
  id: 42,
  title: "Борщ",
  category: "Супы",
  emoji: "🍲",
  portions: 6,
  time: 90,
  photo: "data:image/...",   // base64 фото (или null)
  photoSrc: "https://...",   // URL фото с сайта
  sourceUrl: "https://...",  // ссылка на оригинал (из импорта)
  ingredients: [
    { name: "Свекла", amount: "300", unit: "г" },
    { name: "Для соуса", amount: "", unit: "" }  // ← ЗАГОЛОВОК СЕКЦИИ
  ],
  steps: "Шаг 1: ...\nШаг 2: ...",
  notes: "Подавать со сметаной",
  nutrition: { kcal: 320, protein: 12.5, fat: 8.3, carbs: 45.1, manual: true },
  createdAt: "2026-03-10T..."
}
```

**Заголовки секций:** ингредиент с пустым `amount` И пустым `unit` = заголовок группы. Везде пропускается: `if (!ing.amount && !ing.unit) continue`.

## 📋 Структура слота планировщика

```js
{ day: "Понедельник", note: "Лёгкий день", slots: [
  {
    meal: "Обед",
    items: [
      { type: "recipe",     recipeId: 42, portions: 2 },
      { type: "ingredient", name: "яблоко", amount: 150, unit: "г",
        kcal: 78, protein: 0.4, fat: 0.4, carbs: 19.5, portions: 1 }
    ]
  }
]}
```

**Навигация по неделям:** `plannerWeekOffset` (0=текущая, ±1=соседние). Ключ: `"2026-W10"`.  
**Автомиграция:** при загрузке `ck_planner` → `ck_planner_v2`.

## 🥦 Базы данных продуктов (БЖУ)

### Системная `FOOD_DB` (только чтение)
~600 продуктов, встроена в JS. Формат: `{ "говядина": [187, 18.9, 12.4, 0] }` — [ккал, б, ж, у] на 100г.  
Просмотр: вкладка «База» → «📋 База продуктов программы».

### Пользовательская `userFoodDB` (`ck_user_food_db`)
Тот же формат. Полный CRUD. Приоритет над системной. Экспорт JSON/HTML, импорт мерж.  
Управление: вкладка «База» → «🥦 Моя база продуктов».

### `findFoodEntry(name)` — поиск питательности
Сначала `userFoodDB`, потом `FOOD_DB` (точное → частичное совпадение).

## 🔢 Расчёт БЖУ

- `calcRecipeNutrition(recipe)` → `{ kcal, protein, fat, carbs }` по ингредиентам
- `itemNutr(item)` → нутриенты одного элемента слота
- `slotNutr(slot)` → сумма по всем items
- `plannerNorms` по умолчанию `{ kcal:2000, protein:75, fat:60, carbs:250 }`
- Цвета: ≤90% зелёный · 90–115% оранжевый · >115% красный

## 📋 Быстрый ввод покупок (`modal-shop-quick`)

Кнопка **📋** рядом с полем на вкладке Покупки.

**Поток:**
1. textarea для ввода/вставки
2. «📋 Вставить из буфера» — Clipboard API, fallback: фокус + тост
3. 🎤 — Web Speech API, `lang='ru-RU'`, `continuous=true`. HTTPS only. На `file://` — неактивна.
4. «▶ Разобрать» → превью позиций. Поле очищается, **список накапливается** (не заменяется).
5. ✕ у каждой позиции. 🗑 «Очистить всё». ✅ «Добавить в список».

### Парсер `tokenizeShopText(raw)` — v3, 37/37 тестов

**`precleanLine()`:** невидимые символы, маркеры (`- • * 1.`), заголовки (`Купить:`), глаголы (`купи`, `нужно`).

**`normalizeShopText()`:**
- Слипшиеся единицы: `500г` → `500 г`
- Умножители: `*`, `x`, `×` → `N шт`
- Числительные: `шесть`→`6`, `полтора`→`1.5` (Cyrillic-safe lookaround, не `\b`)
- Единицы → аббревиатуры (26 паттернов)

**Роутинг:**
- `\n` → построчно через `parseShopLine()`
- `,`/`;` → по разделителям
- нет чисел → `parseContinuousShopText()` со словарём (~250 товаров)
- одно число → `parseShopLine()`
- несколько чисел → `parseContinuousShopText()` (машина состояний)

**`parseShopLine()`** — форматы: `Имя ЧИСЛО [ед]` · `ЧИСЛО [ед] Имя` · `Имя — ЧИСЛО [ед]` · просто `Имя`.

**`parseContinuousShopText()`** — сначала словарный поиск `GROCERY_DICT`, потом машина состояний по числам.

**`GROCERY_CANON`** — падежи: `молока`→`молоко`, `яиц`→`яйца` и т.д.

## ⚠️ Подводные камни

1. **Wake Lock** — только HTTPS. На `file://` тихо игнорируется.
2. **CORS-прокси** — нестабильны, перебираются по очереди. При отказе всех — ручной ввод.
3. **russianfood.com шаги** — AJAX, парсер не видит. Workaround: ручной ввод.
4. **Web Share API** — не везде. Fallback: `downloadFile()`.
5. **JSON → Share** — нельзя напрямую, обёртка `<script id="...">` в HTML.
6. **`MEALS` — let** — мутируется при настройке приёмов.
7. **`ck_planner`** — только читается при миграции, рабочий ключ `ck_planner_v2`.
8. **Заголовки секций** — `{ name, amount:'', unit:'' }`, везде `if (!ing.amount && !ing.unit) continue`.
9. **Base64 фото** — лимит localStorage ~5–10 MB.
10. **Числительные-слова** — Cyrillic-safe lookaround `(?<![а-яё])слово(?![а-яё])` вместо `\b`.
11. **🎤 на `file://`** — полупрозрачна, проверка при открытии `modal-shop-quick`.
12. **`openUserFoodDB()`** — обёртка над `openUserFoodDBModal()`, нужна для вызова из пикера.
